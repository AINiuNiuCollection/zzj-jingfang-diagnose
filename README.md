# 张仲景经方辨证辅助技能

## 概述

本技能基于宋版《伤寒论》《金匮要略》原典，为执业中医师提供临床辅助辨证。核心价值是快速匹配原典条文、锁定候选经方、排查方证禁忌、还原仲景辨证思路，按八步流程（含张仲景审计循环）生成论证式可追溯报告。

**定位红线**：辅助参考工具，非独立诊疗系统。最终诊疗决策权归属执业医师。所有输出标注"仅供临床参考"。

## 关键总则
- 所有的流程里必须按照下述“核心工作流程”完成，不得跳过或遗漏，必须客观、中立的深度分析，不回避问题的核心。
- 切记切记：不要头痛医头、脚痛医脚，诊断始终遵循整体到具体，分析问题根因时由表及里、由此及彼抓住关键点；你治的是人而不是病、你调理的是阴阳而不是头痛医头脚痛医脚，最后得出治疗方案。
- 先有总体治疗方案再有具体方剂：方案全面且具体，整体方案论述、当前阶段、剂量（根据患者情况，预估当前阶段治疗周期）、服用时间、煎煮方法、好转标志、变坏标志，什么时候转下一阶段；推荐饮食、按摩穴位、茶饮、运动等等，最后输出完整报告。

## 触发场景

- 用户提供四诊信息，请求经方辨证
- 用户查询伤寒论/金匮要略条文
- 用户请求类方鉴别对比
- 用户提及"六经辨证""方证匹配""仲景""经方"等关键词

## 核心工作流程

> **流程红线**：本流程包含5个强制阻断点（⚠️标记）。命中阻断条件时，**必须使用 AskUserQuestion 工具向用户提问并等待回复**，禁止自行跳过、禁止在追问未回复时继续后续步骤。任何阻断点未通过，后续步骤一律不得执行。

### 第一步：收集四诊信息

向用户收集以下信息。**6项必填缺失时必须阻断追问**，强烈推荐项缺失时在检索后由 info_completeness 机制补问。

**必填核心项**（缺失触发⚠️阻断点1）：
- 主诉：核心辨证起点
- 症状列表：方证匹配主要依据
- 脉象：一级权重，脉证合参核心依据
- 寒热情况：恶寒/恶风/发热/往来寒热/潮热/无——第11条真假寒热鉴别必查项
- 汗出情况：有汗/无汗/汗出不止/自汗/盗汗/不确定——桂枝汤vs麻黄汤分水岭
- 二便：大便（干/溏/不通/下利）、小便（利/不利/清/黄）

**强烈推荐项**（缺失影响辨证精度和剂量安全）：
- 渴饮情况：不渴/渴喜冷饮/渴不欲饮/渴欲饮热——五苓散证vs茯苓甘草汤证鉴别（SHL-073）
- 欲近衣情况：欲近衣/不欲近衣/未知——第11条真假寒热鉴别核心
- 年龄：剂量倍率核心依据（老年→调理量×0.6）
- 体重+身高：BMI计算→极度消瘦→调理量×0.6
- 性别：孕娠禁忌判定
- 精神状态："但欲寐"→少阴（SHL-281）

**选填拓展项**：
- 舌象（原典舌诊一级辅助、后世舌诊二级辅助，详见 `dictionary.json` → `tongue_classical`）
- 人群属性：普通成人/老年体虚/产后经期/儿童/素有宿疾
- 治疗史：未经治疗/发汗后/攻下后/输液后/服用寒凉药后
- **既往病史/素有宿疾**（辅助参考）：如素有痰饮/哮喘/心悸/胃病等。标注"辅助参考信息"，仅作病机分析参考维度，不作为独立辨证依据，不改变"仅以原典为诊疗依据"的红线
- 腹诊/情志/饮食偏好/睡眠/其他补充

**快捷引导**：提供高频症状快捷选项，减少白话表述差异。

---

> **⚠️ 阻断点1：必填项检查**
>
> 收集信息后，立即检查6项必填（主诉/症状/脉象/寒热/汗出/二便）是否齐全。
> - **任一缺失 → 必须使用 AskUserQuestion 工具逐项追问，等待用户回复后才能继续**
> - 追问格式示例：`"脉象为仲景辨证核心依据，请补充脉象信息（如：浮/沉/弦/数/缓/紧/弱/细等）"`
> - 用户明确表示"无法提供脉象"时：**触发脉象缺失兜底方案**，在报告中标注"脉象缺失，辨证精度受限"，并输出脉象缺失兜底参考（按六经脉象纲领列出不同脉象对应的方剂方向），然后继续
> - **脉象缺失兜底方案**：当追问脉象后用户仍无法提供时，检索脚本自动生成 `pulse_fallback` 字段，内容为：
>   - 按六经分类，列出每种可能脉象对应的方剂方向和原典条文依据
>   - 优先展示基于已有症状推断的最可能1-2经的脉象方向
>   - 同时列出全部六经的脉象纲领供参考
>   - 示例：脉浮缓→桂枝汤（SHL-012）；脉沉迟→理中汤辈（SHL-386）；脉弦→小柴胡汤（SHL-096）
> - **禁止在必填项缺失且未追问的情况下直接进入检索**

---

### 第二步：调用检索脚本获取知识库数据

执行检索脚本，获取规则过滤+六经候选+方剂排名+条文召回的结构化结果：

```bash
python scripts/retrieve_kb.py \
  --chief "主诉" \
  --symptoms "症状1,症状2,症状3" \
  --pulse "脉象" \
  --chill-fever "寒热" \
  --sweat "汗出" \
  --stool-urine "二便" \
  --want-clothing "欲近衣" \
  --thirst "渴饮" \
  --population "人群" \
  --treatment-history "治疗史" \
  --extra "其他补充"
```

脚本输出 JSON 结果，包含：
- `status`：ready / emergency / need_inquiry（必填项缺失时返回need_inquiry）
- `emergency`：急重症命中信息（如有）
- `cold_heat`：真假寒热鉴别结论（含 `need_inquiry` 字段）
- `contraindications`：禁忌排除清单
- `six_channel_hypotheses`：六经候选及置信度
- `candidate_formulas`：Top5 候选方剂（含匹配分数）
- `retrieved_clauses`：Top10 召回条文
- `verification_inquiry`：**验证性追问**（通用机制，基于方剂core_indicators自动检测）
- `info_completeness`：**信息完整性检查**（含必填项校验+强烈推荐项按需校验+体质关键项校验）
- `pulse_fallback`：**脉象缺失兜底参考**（脉象为空或用户表示无法提供时生成，按六经脉象纲领列出不同脉象对应的方剂方向+原典条文依据）
- `patient_profile`：**患者体质适配分析**（含体型评估/用药史/微调建议）
- `keywords` / `negations`：检索关键词与否定排除项

**依赖**：脚本需要 `rank_bm25` 和 `jieba`，执行前确认已安装（`pip install rank_bm25 jieba`）。

**编码注意**：Windows 下通过子进程调用此脚本时，请设置环境变量 `PYTHONIOENCODING=utf-8` 确保中文输出正确（否则 `print()` 会按系统默认代码页输出 GBK 编码字节）。

---

> **⚠️ 阻断点2：寒热追问**
>
> 检查检索结果 `cold_heat.need_inquiry` 字段：
> - **need_inquiry=true → 必须使用 AskUserQuestion 工具追问寒热鉴别信息（欲近衣/渴饮/脉象等），等待用户回复后才能继续**
> - 寒热真假未明时，**禁止仅凭表象定寒热治则**（第11条红色规则）
> - 追问格式：使用 `cold_heat.inquiry_points` 中的内容作为追问选项

---

### 第三步：按八步流程执行辨证分析（强制顺序执行，不可跳过、不可合并）

根据检索结果，按仲景八步流程执行综合分析。**每一步必须显式输出该步骤的分析结果，确认无误后再进入下一步**。详细流程参见 `references/eight-step-guide.md`。

八步流程中嵌入3个阻断点：

> **⚠️ 阻断点3：六经追问**（第三步·③抓主症定六经阶段）
>
> 检查 `six_channel_hypotheses` 中Top1六经置信度：
> - **Top1置信度 < 0.65 → 必须使用 AskUserQuestion 工具追问1-2个最能消歧的鉴别点，等待用户回复后才能继续**
> - 追问仍无法消歧时：输出"两经候选并行"并说明各自所需鉴别信息，**禁止强行单经归属**
> - 合病主次不明时：输出"两经并重"并强制追问，**禁止自动合方**

> **⚠️ 阻断点4：验证性追问**（第三步·⑤方证精准匹配阶段）
>
> 检查 `verification_inquiry.need_verification` 字段：
> - **need_verification=true → 必须使用 AskUserQuestion 工具追问1个核心鉴别点，等待用户回复后才能继续**
> - 追问内容使用 `verification_inquiry.inquiry_point` 中的选项
> - 追问目的：避免"高置信度但关键鉴别信息缺失"的误判

> **⚠️ 阻断点5：信息完整性检查**（第三步·八步分析完成后、生成报告前）
>
> 检查 `info_completeness.has_gaps` 字段：
> - **has_gaps=true 且存在 importance=high 的缺口 → 必须使用 AskUserQuestion 工具追问，等待用户回复后才能继续**
> - **has_gaps=true 且仅存在 importance=medium 的缺口 → 不阻断流程，但必须在报告中"待确认事项"栏目逐一列出**
> - 追问内容使用 `info_completeness.gaps[].question`

---

### 第四步：生成 8+1 模块论证式报告（含强制落盘）

输出完整论证式思维链，包括模块0+模块1-8共9个模块（见 `report-template.md`），禁止省略，做到"立与破皆有据"。

**模块0（总体治疗方案）必须在模块1-8之前首先输出**，给患者和医师完整的治疗全貌。模块0包含：整体治疗策略论述、治疗阶段规划表（含所有阶段方剂方向/治则/周期/转方标准）、当前所处阶段、各阶段逻辑关系与原典依据。

**信息缺口标注要求**：若阻断点5中存在未补齐的 medium 级缺口，必须在报告模块1末尾增加"**待确认事项**"子节，逐条列出缺口字段、影响说明和建议追问内容。

### 4.1 保存路径
- 目录：`{当前技能运行目录}/reports/`
- 命名：`patient_{性别}{年龄}_{疾病简名}_{YYYYMMDD}_{随机4为编位}.md`
- 示例：`patient_M31_IgA_20260712_0001.md`

### 4.2 执行方式（三选一，推荐第1种）
1. **write 工具**（跨平台，推荐）：直接调用 write 工具写入文件
2. **bash 重定向**：`cat > reports/patient_M31_IgA_20260712_0001.md << 'EOF' ... EOF`

### 4.3 验证
- 保存后立即检查文件是否存在且大小 > 0
- 向用户返回**绝对文件路径**，供用户直接打开

### 4.4 失败处理
- 保存失败时：报错、重试、或提示用户手动复制
- 严禁以"已在对话中输出"为由跳过落盘

### 条文查询（独立功能）

### 类方鉴别（独立功能）

用户请求类方对比时，从 `assets/formulas.json` 中提取相关方剂数据，按以下维度对比：
- 核心指征 / 排除指征 / 方剂组成 / 主治主症 / 典型脉象 / 核心病机 / 禁忌 / 鉴别要点 / 剂量换算

鉴别指南参见 `references/formula-differentiation.md`。

## 关键约束（红线）

1. **仅以宋版原典为唯一诊疗依据**，不混入后世医家经验
2. **原典禁忌场景 100% 拦截**（红色一票否决），无方证用反、治则颠倒；十八反十九畏为后世安全规则（橙色），有仲景原典先例者降权30%，无先例者降权50%，不直接否决
3. **真假寒热鉴别必查**（依据第11条），未明不定寒热治则
4. **所有结论 100% 可溯源**至原典条文，无条文支撑的内容禁止输出
5. **信息不全时不强行结论**，触发智能追问（含六经追问+验证性追问双机制）
6. **脉证合参**（非后世"舍证从脉"），脉证冲突触发多假设追踪
7. **规则版本化管理**，每次报告标注所用规则版本号
8. **输出强制免责声明**
9. **后世补充信息须标注来源层级**，不冒充原典
10. **舌诊分级处理**：原典舌诊（仲景原文直接记载，如舌上白苔滑、舌黄、唇痿舌青等10项，见 `dictionary.json` → `tongue_classical`）为一级辅助，与脉象同级参考；后世舌诊（如舌淡苔白滑→寒、舌红苔黄燥→热等系统化舌诊）为二级辅助，不作为独立辨证依据，须标注"后世补充"
11. **整体观原则**：从整体到局部，治人非治病、理阴阳而非头痛医头，治疗目标为恢复阴阳平衡、气血充盈。整体策略论述须引用原典治则条文
12. **预后标志可溯源**：好转/变坏/转阶段标志均须引用原典预后条文，不得凭空推测
13. **既往病史仅作辅助参考**：患者病史信息标注"辅助参考"层级，不作为独立辨证依据，不改变原典溯源承诺
14. **药性提示基于原典组成**：药性判断以原典方剂组成中的药味为依据，不引入后世药性理论扩展
15. **方剂微调须有原典依据**：患者个体化调整必须在仲景原典条文范围内，不得引入后世加减法。所用微调须标注对应条文编号，无条文支撑的调整禁止输出
16. **张仲景审计强制执行**：经方微调后必须执行张仲景审计流程（评分≥90通过），不通过时循环回退至第五步半重新推演，最多3轮。审计评分纳入报告元信息

## 剂量换算标准

采用柯雪帆/郝万山考证：**汉代一两 ≈ 15.625g**

- 常规调理量：原量 × 0.6
- 常规治疗量：原量 × 1.0
- 急证重证量：原量 × 2.0

争议透明：通行教材"一两≈3g"为保守参考，日本汉方"一两≈1g"仅作对照。最终剂量由执业医师核定。

## 已知局限（须告知用户）

1. 仅擅长典型经方证匹配，症状极不典型时匹配度下降
2. 方剂微调仅限于仲景原典框架内的条文推演（如SHL-096加减法），不包括后世医家加减经验
3. 不包含后世医家经验，仅代表仲景原典体系
4. 舌诊等原典稀疏维度，部分判断需参考后世补充（已标注层级）
5. 仅覆盖高频误治变证，复杂坏病暂不支持
6. 剂量换算存在学界争议，最终剂量须执业医师核定
7. 药性适配分析仅基于原典方剂组成药味，不引入后世药性归经理论扩展
8. 体质适配分析（如BMI、激素用药史）标注"辅助参考"层级，不作为独立辨证依据
