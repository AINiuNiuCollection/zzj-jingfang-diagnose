# 张仲景经方辨证辅助技能

## 概述

本技能基于宋版《伤寒论》《金匮要略》原典，为执业中医师提供临床辅助辨证。核心价值是快速匹配原典条文、锁定候选经方、排查方证禁忌、还原仲景辨证思路，按八步流程（含张仲景审计循环）生成论证式可追溯报告。

**定位红线**：辅助参考工具，非独立诊疗系统。最终诊疗决策权归属执业医师。所有输出标注"仅供临床参考"。

## 重要-总体要求
- 所有的流程里必须按照下述“核心工作流程”完成，不得跳过或遗漏，必须客观、中立的深度分析，不回避问题的核心。
- 切记切记：不要头痛医头、脚痛医脚，始终诊断应先遵循整体到具体，需分析问题根因由表及里、由此及彼，你治的是人而不是病、你调理的是阴阳而不是头痛医头脚痛医脚，最后得出结论开方。
- 应该先有总体治疗方案：方案全面且具体，整体方案论述，当前阶段、剂量（根据患者情况，预估当前阶段治疗周期）、服用时间、煎煮方法、好转标志、变坏标志，什么时候转下一阶段；推荐饮食、按摩穴位、茶饮、运动等等，最后给出总结


## 触发场景

- 用户提供四诊信息，请求经方辨证
- 用户查询伤寒论/金匮要略条文
- 用户请求类方鉴别对比
- 用户提及"六经辨证""方证匹配""仲景""经方"等关键词

## 核心工作流程

### 第一步：收集四诊信息

向用户收集以下信息（不强制填写，但标注信息不全可能影响辨证准确性）：

**必填核心项**：
- 寒热情况：恶寒/恶风/发热/往来寒热/潮热/无
- 汗出情况：有汗/无汗/汗出不止/自汗/盗汗/不确定
- 脉象：如脉浮紧、脉浮缓、脉弦等
- 二便：大便（干/溏/不通/下利）、小便（利/不利/清/黄）

**选填拓展项**：
- 舌象（二级辅助，原典舌诊稀疏）
- 欲近衣情况（真假寒热鉴别关键）：欲近衣/不欲近衣/未知
- 渴饮情况：不渴/渴喜冷饮/渴不欲饮/渴欲饮热
- 人群属性：普通成人/老年体虚/产后经期/儿童/素有宿疾
- 治疗史：未经治疗/发汗后/攻下后/输液后/服用寒凉药后
- **既往病史/素有宿疾**（辅助参考）：如素有痰饮/哮喘/心悸/胃病等。标注"辅助参考信息"，仅作病机分析参考维度，不作为独立辨证依据，不改变"仅以原典为诊疗依据"的红线

**快捷引导**：提供高频症状快捷选项，减少白话表述差异。信息极度不全时（2-3向没有），主动追问核心鉴别信息。

**v2新增·信息完整性自动检查**：检索脚本输出 `info_completeness` 字段，在报告生成前系统检查信息缺口：
- 🔴 **脉象缺失**：脉诊为仲景辨证一级依据，缺失则六经定性与方证匹配均会降权，强制追问
- 🟡 **Top1候选方所需鉴别信息缺失**：如泻心汤类缺少腹部触诊信息（痞/结胸鉴别）、利水方缺少小便细节
- 🟡 **特殊体质信息缺口**：如极低BMI缺少体重变化史、长期用药史缺少具体方案
- 以上检查结果作为报告生成前置步骤，信息缺口在报告中以"待确认事项"列出

### 第二步：调用检索脚本获取知识库数据

执行检索脚本，获取规则过滤+六经候选+方剂排名+条文召回的结构化结果：

```bash
python scripts/retrieve_kb.py \
  --chief "主诉" \
  --symptoms "症状1,症状2,症状3" \
  --pulse "脉象" \
  --chill-fever "寒热" \
  --sweat "汗出" \
  --stool-urine "二便" \
  --want-clothing "欲近衣" \
  --thirst "渴饮" \
  --population "人群" \
  --treatment-history "治疗史" \
  --extra "其他补充"
```

脚本输出 JSON 结果，包含：
- `status`：ready / emergency / need_inquiry
- `emergency`：急重症命中信息（如有）
- `cold_heat`：真假寒热鉴别结论
- `contraindications`：禁忌排除清单
- `six_channel_hypotheses`：六经候选及置信度
- `candidate_formulas`：Top5 候选方剂（含匹配分数）
- `retrieved_clauses`：Top10 召回条文
- `verification_inquiry`：**验证性追问**（v2覆盖6类方证，见下方说明）
- `info_completeness`：**信息完整性检查**（v2新增，系统检查四诊信息缺口）
- `patient_profile`：**患者体质适配分析**（v2新增，含体型评估/用药史/微调建议）
- `keywords` / `negations`：检索关键词与否定排除项

**依赖**：脚本需要 `rank_bm25` 和 `jieba`，执行前确认已安装（`pip install rank_bm25 jieba`）。

**编码注意**：Windows 下通过子进程调用此脚本时，请设置环境变量 `PYTHONIOENCODING=utf-8` 确保中文输出正确（否则 `print()` 会按系统默认代码页输出 GBK 编码字节）。

### 第三步：按七步流程执行辨证分析

根据检索结果，按仲景七步流程执行综合分析。详细流程参见 `references/seven-step-guide.md`。

#### 第零步：治则预判（急重症+真假寒热）

**急重症预判**：检查检索结果 `emergency` 字段。若命中急重症，立即优先执行急救治则，红色高亮警示，不进入常规流程。详见 `references/emergency-checklist.md`。

**真假寒热鉴别**：检查 `cold_heat` 字段。若 `need_inquiry=true`，触发追问，不定寒热治则。详见 `references/cold-heat-rules.md`。

**禁忌拦截**：检查 `contraindications` 字段。红色排除的方剂不进入候选。详见 `references/contraindication-rules.md`。

#### 第一步：信息标准化

检索脚本已完成术语映射（白话→文言）和否定提取。确认 `normalized_query` 和 `negations` 字段。

#### 第二步：抓主症定六经

检查 `six_channel_hypotheses`。多假设追踪规则：
- Top1 置信度 ≥ 0.65：可确定主经
- 置信度 < 0.65 且主假设不明显领先：触发追问消歧
- 两经置信度接近（差<0.2）：判定合病，依合病决策表定主次

#### 第三步：误治史适配

若用户有治疗史，匹配误治变证条文（覆盖范围见 `assets/mistreatment.json`）。清单外场景明示暂不支持。

#### 第四步：方证匹配与验证性追问

检查 `candidate_formulas`。每首方剂的匹配依据：
- `core_hits`：特异性核心指征命中数（最高权重，一票定方）
- `main_hits`：主症命中数
- `pulse_hits`：脉象命中数
- 排除指征命中 → 直接剔除

按匹配度排序 Top3。方证鉴别要点参见 `references/formula-differentiation.md`。


### 第三步：按七步流程执行辨证分析（强制顺序执行，不可跳过、不可合并）

根据检索结果，按仲景七步流程执行综合分析。**每一步必须显式输出该步骤的分析结果，确认无误后再进入下一步**。详细流程参见 `references/seven-step-guide.md`。

#### 第零步：治则预判（急重症+真假寒热）

**急重症预判**：检查检索结果 `emergency` 字段。若命中急重症，立即优先执行急救治则，红色高亮警示，不进入常规流程。详见 `references/emergency-checklist.md`。

**真假寒热鉴别**：检查 `cold_heat` 字段。若 `need_inquiry=true`，触发追问，不定寒热治则。详见 `references/cold-heat-rules.md`。

**禁忌拦截**：检查 `contraindications` 字段。红色排除的方剂不进入候选。详见 `references/contraindication-rules.md`。

> **执行要求**：输出「急重症结论」「寒热鉴别结论」「禁忌拦截清单」三项。

#### 第一步：信息标准化

检索脚本已完成术语映射（白话→文言）和否定提取。确认 `normalized_query` 和 `negations` 字段。

> **执行要求**：输出「标准化后症状列表」「否定排除项」。

#### 第二步：抓主症定六经

检查 `six_channel_hypotheses`。多假设追踪规则：
- Top1 置信度 ≥ 0.65：可确定主经
- 置信度 < 0.65 且主假设不明显领先：触发追问消歧
- 两经置信度接近（差<0.2）：判定合病，依合病决策表定主次

> **执行要求**：输出「六经假设列表(含置信度)」「主经归属」「合病判定(如有)」「追问需求(如有)」。

#### 第三步：误治史适配

若用户有治疗史，匹配误治变证条文（覆盖范围见 `assets/mistreatment.json`）。清单外场景明示暂不支持。

> **执行要求**：输出「误治变证匹配结果」或「无治疗史/不适用」。

#### 第四步：方证匹配与验证性追问

检查 `candidate_formulas`。每首方剂的匹配依据：
- `core_hits`：特异性核心指征命中数（最高权重，一票定方）
- `main_hits`：主症命中数
- `pulse_hits`：脉象命中数
- 排除指征命中 → 直接剔除

按匹配度排序 Top3。方证鉴别要点参见 `references/formula-differentiation.md`。

**验证性追问（v2增强版）**：候选方Top1确定后，系统检查6类方证场景的关键鉴别信息是否充分。即使置信度达标，若Top1方的核心鉴别指征缺失，触发验证性追问，仅追问1个最能验证Top1适用性的鉴别点。

> **执行要求**：输出「Top3候选方(含匹配分数/命中详情)」「验证性追问结果(触发/未触发)」。

#### 第五步：禁忌二次复核

对 Top3 候选方逐一复核：
- 方剂自身禁忌是否命中
- 有毒药材超量拦截
- 人群禁忌适配
- **药性适配复核**：检查候选方药性与患者体质是否冲突（寒凉方剂用于脾虚患者→黄色警示；温燥方剂用于阴虚患者→黄色警示），基于方剂组成药味判断
- 标注风险等级（红/黄/蓝）

> **执行要求**：输出「Top3方剂禁忌复核表(含风险等级)」。

#### 第五步半：经方合理微调 — 基于原典的患者适配调整

**此步骤必须在生成报告前单独执行**，合并到报告中。

**红线**
- 以经方为主加减，讲究药简力专，以系统、整体思维治疗人，非治疗局部的病
- 一定基于患者历史病况主要考虑结合当前病情考虑加减原因
- 药性需考虑当代人工种植情况，适当加减用量
- 最后成方需要有临床过中医经典根据（包括不限于黄帝内经、神农本草经、伤寒论、金匮要略）
- 寒凉、热燥药物都要结合患者实际情况综合考虑而定
- 切记切记：不要头痛医头、脚痛医脚，要从整体出发到局部，你治的是人、理的是阴阳而不是病，目的是让整个人精气神恢复、气血充盈、阴阳平衡
- 所有微调必须标注原典条文编号，不可引入后世经验。最终处方由执业医师核定。

检索脚本输出 `patient_profile` 字段，输出三类微调建议（均标注原典依据）：

**① 体质剂量调整**：
- 极度消瘦（BMI<17）、年老体虚：建议调理量（×0.6）起始，峻猛药（大黄、芒硝、麻黄等）减量或去之
- 依据：SHL-273"太阴之为病...食不下"，虚人宜缓图

**② 原典加减法推演**：
在仲景原典条文范围内，请根据实际情况在符合仲景药力专、随证治之前提下进行，根据患者个体差异可推演的合理加减：
分析示例：
| 患者表现 | 可考虑加减 | 原典依据 |
|:---|:---|:---|
| 小便不利/水入不化 | 加茯苓 | SHL-096"心下悸、小便不利者，去黄芩加茯苓" |
| 心烦易怒 | 加黄芩/栀子 | SHL-096"心烦喜呕"用黄芩；SHL-076栀子豉汤 |
| 脾虚食不化 | 加人参（或党参） | SHL-386理中丸治中虚用人参 |
| 咳 | 加五味子、干姜 | SHL-096"咳者，去人参大枣生姜加五味子干姜" |
| 腹中痛 | 加芍药 | SHL-096"腹中痛者，去黄芩加芍药" |
| 渴 | 加栝蒌根 | SHL-096"渴者，去半夏加人参栝蒌根" |

**③ 合方推演**：
当患者证候跨越两方主证时，可参考合病治法决策表（`assets/hebian.json`）推演合方。合方的原典依据必须明确可溯（如柴胡桂枝汤SHL-146为太阳少阳合病之例）。

> **执行要求**：输出「体质剂量调整建议」「原典加减法推演表」「合方推演(如适用)」。

#### 第六步（新增）：张仲景审计（仲景评分）

经方微调完成后，**必须进入张仲景审计环节**。以"张仲景"角色视角，对当前形成的完整治疗方案与处方进行独立评审。评审细则参见 `references/zzj-audit-guide.md`。

**审计维度**（每项独立打分，满分100）：

| 维度 | 权重 | 审视要点 |
|:---|:---:|:---|
| ① 整体观 | 20分 | 是否从整体到局部、治人非治病、理阴阳非头痛医头？根因分析是否由表及里？ |
| ② 原典依据 | 20分 | 每味药、每个结论是否可溯源至经典（伤寒论/金匮/内经/本经）？无原典支撑即扣分 |
| ③ 方证匹配度 | 15分 | 主方与患者核心病机是否高度对应？核心指征命中率？非以症状数量定匹配 |
| ④ 加减合理性 | 15分 | 微调是否在仲景原典加减法范围内？有无引入后世经验？加减理由是否充分 |
| ⑤ 体质适配 | 10分 | 剂量是否考虑体质（BMI/年龄/虚实）？峻猛药是否斟酌减量？ |
| ⑥ 药性平衡 | 10分 | 寒热温凉是否与患者体质协调？有无纯寒纯温偏颇之弊？ |
| ⑦ 治疗方案完整性 | 10分 | 治疗阶段、煎服法、预后标志、调护、饮食、穴位、运动是否完整？ |

**评分与循环规则**：
- **≥ 90分**：审计通过 ✅ → 进入第七步（生成报告+强制落盘）
- **< 90分**：审计不通过 ❌ → **必须回退至第五步半（经方合理微调）**，结合张仲景的评审意见重新推演微调方案，调整后再次进入审计，直至评分≥90

**循环记录要求**：
- 每次循环需标注迭代次数（如「张仲景审计·第2轮」）
- 每次需说明针对上一轮评审意见做了哪些具体修改
- 禁止无限循环：最多迭代3次。第3次仍未通过则输出全部审计历史，由执业医师人工裁断

> **执行要求**：输出「仲景评分」「各维度得分明细」「评审意见与改进建议」「审计结论（通过/不通过+循环指引）」「迭代次数标记」。评分<90分时强制返回第五步半重新推演并标注循环号。


### 第五步：生成 8 模块论证式报告（含强制落盘）
包括但不限于按 8 模块标准生成报告。模板参见 `references/report-template.md`。
**强制落盘要求**：生成报告后，**必须立即调用 write 工具保存到文件**，不得仅在对话中输出。
- 所有输出信息都要保存到输出文件，包括但不限于下述1-8条信息
- 保存路径：`./reports/patient_{性别}{年龄}_{疾病简名}_{YYYYMMDD}_{随机4位编号}.md`
- 示例：`patient_M31_IgA_20260712-001.md`
- 保存后验证文件存在且大小 > 0，向用户返回**绝对文件路径**

1. **核心信息提取**：主诉、症状、脉象（一级）、舌象（二级辅助）、关键鉴别信息、既往病史（辅助参考）、系统识别的核心辨证要点
2. **六经辨证结论**：主经归属、多假设置信度、合病判定、排除其他经依据、寒热鉴别结论
3. **候选方剂推荐 Top3**：方名、出处条文、组成、剂量换算（柯雪帆标准）、标准加减、匹配依据、易混方排除原因
4. **煎服与调护**：煎煮方法（古今换算）、服用方式（含原典服药周期）、药后调护、停药指征、饮食禁忌（原典记载）、**预后观察标志**（好转标志✅/变坏标志⚠️/阶段转换标准🔄，均引用原典预后条文）、药材质量提示
5. **风险与禁忌提示**：分级标注（🔴红/🟡黄/🔵蓝）、有毒药材安全范围、**药性提示**（寒凉/温燥药性与患者体质适配性）
6. **治则说明**：治疗原则、先后顺序、**整体治疗策略**（治人非治病、理阴阳，引用原典治则条文）、真假寒热鉴别结论与依据、患者病史参考（如有）
7. **原文溯源**：每条结论对应条文编号，可跳转查看原文
8. **可追溯元信息**：规则版本号（v1.0.0）、Top-K 条文编号、置信度评分、触发规则列表

**报告末尾强制展示免责声明**：
> 本结果仅为经方学术参考，不构成诊疗建议，具体处方请由执业医师辨证开具。

## 关键约束（红线）

1. **仅以宋版原典为唯一诊疗依据**，不混入后世医家经验
2. **禁忌场景 100% 拦截**（一票否决），无方证用反、治则颠倒
3. **真假寒热鉴别必查**（依据第11条），未明不定寒热治则
4. **所有结论 100% 可溯源**至原典条文，无条文支撑的内容禁止输出
5. **信息不全时不强行结论**，触发智能追问（含六经追问+验证性追问双机制）
6. **脉证合参**（非后世"舍证从脉"），脉证冲突触发多假设追踪
7. **规则版本化管理**，每次报告标注所用规则版本号
8. **输出强制免责声明**
9. **后世补充信息须标注来源层级**，不冒充原典
10. **舌诊降为二级辅助**（原典舌诊稀疏），不泛化舌象-方剂关联
11. **整体观原则**：从整体到局部，治人非治病、理阴阳而非头痛医头，治疗目标为恢复阴阳平衡、气血充盈。整体策略论述须引用原典治则条文
12. **预后标志可溯源**：好转/变坏/转阶段标志均须引用原典预后条文，不得凭空推测
13. **既往病史仅作辅助参考**：患者病史信息标注"辅助参考"层级，不作为独立辨证依据，不改变原典溯源承诺
14. **药性提示基于原典组成**：药性判断以原典方剂组成中的药味为依据，不引入后世药性理论扩展
15. **方剂微调须有原典依据**：患者个体化调整必须在仲景原典条文范围内，不得引入后世加减法。所用微调须标注对应条文编号，无条文支撑的调整禁止输出
16. **张仲景审计强制执行**：经方微调后必须执行张仲景审计流程（评分≥90通过），不通过时循环回退至第五步半重新推演，最多3轮。审计评分纳入报告元信息

## 剂量换算标准

采用柯雪帆/郝万山考证：**汉代一两 ≈ 15.625g**

- 常规调理量：原量 × 0.6
- 常规治疗量：原量 × 1.0
- 急证重证量：原量 × 2.0

争议透明：通行教材"一两≈3g"为保守参考，日本汉方"一两≈1g"仅作对照。最终剂量由执业医师核定。

## 已知局限（须告知用户）

1. 仅擅长典型经方证匹配，症状极不典型时匹配度下降
2. 方剂微调仅限于仲景原典框架内的条文推演（如SHL-096加减法），不包括后世医家加减经验

6. 剂量换算存在学界争议，最终剂量须执业医师核定
7. 药性适配分析仅基于原典方剂组成药味，不引入后世药性归经理论扩展
8. 体质适配分析（如BMI、激素用药史）标注"辅助参考"层级，不作为独立辨证依据
