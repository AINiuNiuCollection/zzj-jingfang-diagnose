# 仲景八步辨证流程详解

> 本文档指导 AI 按仲景诊疗逻辑执行辨证，含张仲景审计循环。基础八步流程 + 新增第六步（张仲景审计）。

## 总原则

严格遵循仲景"先定治则、再辨病位、最后选方"的原生思路，拒绝线性症状权重叠加，全程透明可复核。

**治则优先原则**：治则优先级 > 病位判定 > 方证匹配，绝不允许以方废则。

**整体观原则**：诊断从整体到具体、由表及里、由此及彼，治的是人而非病，调理的是阴阳而非头痛医头脚痛医脚。最终目的是让人精气神恢复、气血充盈、阴阳平衡。所有辨证结论须体现整体思维，引用原典治则条文为依据（如第16条"观其脉证，知犯何逆，随证治之"）。

## 第一步：治则优先级与急重症预判（含真假寒热鉴别）

### 1.1 急重症预判（最高优先级）
逐项对照急重症判定清单（见 `emergency-checklist.md`），命中即优先执行急救治则，红色高亮警示。
- 少阴三急下证 → 大承气汤急下存阴
- 蛔厥 → 乌梅丸安蛔止痛
- 蓄血如狂/发狂 → 抵当汤/桃核承气汤破血逐瘀
- 脏结 → 标注难治，提示转诊
- 亡阳证 → 四逆汤辈急温回阳
- 结胸热实 → 大陷胸汤泻热逐水

### 1.2 真假寒热鉴别（必查项，依据原典第11条）
检查 `cold_heat` 字段。若 `need_inquiry=true`，触发追问，不定寒热治则。详见 `references/cold-heat-rules.md`。
原典第11条："病人身大热，反欲得近衣者，热在皮肤，寒在骨髓也；身大寒，反不欲近衣者，寒在皮肤，热在骨髓也。"

判定要点：
1. 寒热表象与"欲/不欲近衣"是否一致
2. 口渴与饮水情况（真热渴喜冷饮、真寒假热口渴不欲饮或欲饮热）
3. 脉象是否与寒热表象一致（真寒假热脉沉微、真热假寒脉滑数有力）
4. 小便清长与否、舌象真假

**红色规则**：寒热真假未明时，禁止仅凭表象定寒热治则，触发智能追问至鉴别信息充足。

### 1.3 治则优先级判定
- 表里同病：下利清谷者，急当救里，禁先解表（第91条）
- 表里同病：表证为主、里证轻者，先解表后治里
- 虚实夹杂：虚人患病，先扶正后祛邪，禁峻攻
- 急重症优先：三急下证、亡阳证等，优先执行急救治则

### 1.4 禁忌拦截
检查 `contraindications` 字段。红色排除的方剂不进入候选。详见 `references/contraindication-rules.md`。
输出「方剂排除清单」（红色拦截）+「低优先级清单」（黄色降权），约束后续检索范围。

## 第二步：信息标准化处理

接收用户输入的四诊、治疗史、人群属性信息，通过术语映射词典转换为仲景体系标准术语。

拆分为五类：
- **主症**（一级权重）：核心辨证依据
- **兼症**（二级权重）：辅助参考
- **脉象**（一级权重）：仲景核心，脉证合参
- **舌象**（二级辅助）：原典舌诊稀疏，降级处理
- **否定症状**：生成排除清单，前置排除检索

## 第三步：抓主症，定六经病位（含多假设追踪）

### 3.1 提取核心要素
提取3-5个核心主症+关键脉象，判定核心六经归属。

### 3.2 多假设追踪
临床早期信息常不足以确定单一六经归属（如"头痛发热"可见于太阳/阳明/少阳），系统维持Top2-3候选六经并行，每假设带置信度评分。

### 3.3 置信度阈值机制
- Top1置信度 < 0.65 时，触发智能追问最能消歧的1-2个鉴别点
- 追问仍无法消歧时，输出"两经候选并行"并说明各自所需鉴别信息
- 禁止强行单经归属

### 3.4 合病判定
存在多经症状时，依据合病治法决策表判定主次关系：
示例
- 太阳少阳合病 → 柴胡桂枝汤（第146条）
- 太阳阳明合病下利 → 葛根汤（第32条）
- 少阳阳明合病 → 大柴胡汤（第103/136条）
- 三阳合病 → 白虎汤类（第219条）
- 主次不明 → 输出"两经并重"并强制追问，禁止自动合方

## 第四步：误治史适配校验

若用户填写了治疗史/误治史，优先匹配误治变证相关条文：
- 发汗后 → 汗后表虚/汗漏不止（第20条桂枝加附子汤证）
- 攻下后 → 下后痞证/结胸/下利（第154条、第34条）
- 火逆证 → 第114-119条
- 烧针亡阳 → 第117/118条

**明示边界**：复杂坏病、多次误治叠加变证暂不支持。

## 第五步：方证精准匹配

检查 `candidate_formulas`。每首方剂的匹配依据：
- `core_hits`：特异性核心指征命中数（最高权重，一票定方）
- `main_hits`：主症命中数
- `pulse_hits`：脉象命中数
- 排除指征命中 → 直接剔除

按匹配度排序 Top3。方证鉴别要点参见 `references/formula-differentiation.md`。

**验证性追问（v2增强版）**：候选方Top1确定后，系统检查6类方证场景的关键鉴别信息是否充分。即使置信度达标，若Top1方的核心鉴别指征缺失，触发验证性追问，仅追问1个最能验证Top1适用性的鉴别点。

> **执行要求**：输出「Top3候选方(含匹配分数/命中详情)」「验证性追问结果(触发/未触发)」。

### 5.1 验证性追问（方证匹配后增强环节）

候选方剂确定后，检查该方剂的**关键鉴别信息**是否充分。即使置信度达标，若候选方的核心鉴别指征相关信息仍缺失，触发验证性追问：

- **追问触发条件**：候选方Top1的排除指征相关信息未明确（如桂枝汤为Top1但"汗出情况"未填写——桂枝汤与麻黄汤的分水岭正是有汗/无汗）
- **追问内容**：仅追问1个最能验证Top1方剂适用性的鉴别点，采用快捷选项形式
- **追问目的**：在方证匹配后增加一道验证关卡，避免"高置信度但关键鉴别信息缺失"的误判
- **与现有追问的区别**：现有追问在第二步（六经归属置信度不足时触发），验证性追问在第四步后（方剂已匹配但关键鉴别信息缺失时触发），两者互补不冲突

示例：Top1=桂枝汤（核心指征含"汗出"），但用户未填写汗出情况 → 追问"请确认有无汗出：□有汗 □无汗"

## 第六步：禁忌二次复核

对候选方剂逐一复核：
- 确认无方证不符
- 确认无用药禁忌
- 标注风险等级（红色绝对禁忌/黄色相对禁忌/蓝色注意事项）
- 有毒药材超量红色拦截
- **药性适配复核**：检查候选方剂药性与患者体质是否冲突（如寒凉方剂用于脾胃虚弱患者→黄色警示；温燥方剂用于阴虚患者→黄色警示），基于方剂组成中的药味判断，不引入后世药性理论

## 第七步：经方合理微调 — 基于原典的患者适配调整

**此步骤必须在生成报告前单独执行**，合并到报告中。

**红线**
- 切记切记：不要头痛医头、脚痛医脚，要从整体出发到局部，你治的是人、理的是阴阳而不是病，目的是让整个人精气神恢复、气血充盈、阴阳平衡
- 以经方为主加减，以系统、整体思维治疗人，非治疗局部的病
- 讲究药简力专：以系统、整体思维治疗人，总体好转则局部问题改善，严禁总药味数大于13
- 一定基于患者历史病况主要考虑结合当前病情考虑加减原因
- 药性需考虑当代人工种植情况，适当加减用量
- 最后成方需要有临床过中医经典根据（包括不限于黄帝内经、神农本草经、伤寒论、金匮要略）
- 寒凉、热燥药物都要结合患者实际情况综合考虑而定
- 所有微调必须标注原典条文编号，不可引入后世经验，最终处方由执业医师核定。

检索脚本输出 `patient_profile` 字段，输出三类微调建议（均标注原典依据）：

**① 体质剂量调整**：
- 极度消瘦（BMI<17）、年老体虚：建议调理量（×0.6）起始，峻猛药（大黄、芒硝、麻黄等）减量或去之
- 依据：SHL-273"太阴之为病...食不下"，虚人宜缓图

**② 原典加减法推演**：
在仲景原典条文范围内，请根据实际情况在符合仲景药力专、随证治之前提下进行，根据患者个体差异可推演的合理加减：
分析示例：
| 患者表现 | 可考虑加减 | 原典依据 |
|:---|:---|:---|
| 小便不利/水入不化 | 加茯苓 | SHL-096"心下悸、小便不利者，去黄芩加茯苓" |
| 心烦易怒 | 加黄芩/栀子 | SHL-096"心烦喜呕"用黄芩；SHL-076栀子豉汤 |
| 脾虚食不化 | 加人参（或党参） | SHL-386理中丸治中虚用人参 |
| 咳 | 加五味子、干姜 | SHL-096"咳者，去人参大枣生姜加五味子干姜" |
| 腹中痛 | 加芍药 | SHL-096"腹中痛者，去黄芩加芍药" |
| 渴 | 加栝蒌根 | SHL-096"渴者，去半夏加人参栝蒌根" |

**③ 合方推演**：
当患者证候跨越两方主证时，可参考合病治法决策表（`assets/hebian.json`）推演合方。合方的原典依据必须明确可溯（如柴胡桂枝汤SHL-146为太阳少阳合病之例）。

> **执行要求**：输出「体质剂量调整建议」「原典加减法推演表」「合方推演(如适用)」。

## 第八步（新增）：张仲景审计（仲景评分）

**此步骤在第七步（经方合理微调）之后、生成报告之前强制执行**，不可跳过。

### 8.1 审计角色

以"张仲景"视角进行独立评审——熟悉现代医疗方法，也熟悉运用《伤寒论》《金匮要略》《黄帝内经》《神农本草经》等经典医学著作，强项是结合望闻问切和现代医院检测手段，按古代经典开方剂。

### 8.2 评审流程

```
治疗方案和方剂 → 论证患者情况 → 论证结果 → 评审结论 → 评分
```

### 8.3 审计维度与评分标准
**红线** 方剂用药数量超过13味直接0分
| 维度 | 满分 | 审视要点 |
|:---|:---:|:---|
| ① 整体观 | 20 | 从整体到具体、由表及里、治人非治病、理阴阳非头痛医头；根因分析是否触及核心病机 |
| ② 原典依据 | 20 | 所有结论和药味是否可溯源至经典（伤寒论/金匮/内经/本经），无原典支撑即扣分 |
| ③ 方证匹配度 | 15 | 主方与核心病机高度对应，核心指征命中率高；不以症状数量定匹配 |
| ④ 加减合理性 | 15 | 微调在仲景原典加减法范围内，非后世经验；加减理由充分且标注条文 |
| ⑤ 体质适配 | 10 | 剂量考虑BMI/年龄/虚实；峻猛药减量或去之 |
| ⑥ 药性平衡 | 10 | 寒热温凉与患者体质协调；无纯寒纯温偏颇 |
| ⑦ 治疗方案完整性 | 10 | 当前阶段剂量、煎服法、好转/变坏标志、转阶段标准、调护建议完整 |

### 8.4 评分与循环

- **≥ 90分**：通过 ✅ → 进入第七步（生成报告）
- **< 90分**：不通过 ❌ → **回退至第七步**，结合评审意见重新推演微调
- 最多迭代3次，第3次仍未通过则输出全部审计历史，由执业医师人工裁断

### 8.5 循环记录要求

每次循环输出：
- 迭代次数标记（如「张仲景审计·第2轮」）
- 上一轮评审意见的回应与修改说明
- 新一轮评分与结论

## 关键推理规则

1. **治则优先原则**：治则优先级 > 病位判定 > 方证匹配
2. **特异性指征最高权重**：核心指征命中直接升权，排除指征命中直接剔除
3. **脉证合参**：脉象与症状冲突时，严格遵循仲景原典"脉证合参、互为佐证"的原典逻辑，不做简单加权平均，亦不套用后世"舍证从脉/舍脉从证"的二元决策。脉证不一致时触发多假设追踪。
4. **主症决定核心方**：不以症状数量多少定匹配度，而以仲景原文的"核心主症"为第一匹配依据
5. **合病分主次**：多经合病时，依据合病决策表明确核心病位与次要病位
6. **多假设追踪**：六经归属与方证匹配全程维持多假设并行，直到信息充分消歧
